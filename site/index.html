  • "get" test: Populate a map with a pre-generated set of keys (in the JMH setup), make ~50% successful and ~50% unsuccessful "get" calls. For non-identity maps with object keys a distinct set of keys is used (the different object with the same value is used for successful "get" calls).
  • "put/update" test: Add a pre-generated set of keys to the map. In the second loop add the equal set of keys (different objects with the same values) to this map again (make the updates). Identical keys are used for identity maps and for maps with primitive keys.
  • "put/remove" test: In a loop: add 2 entries to a map, remove 1 of existing entries ("add" pointer is increased by 2 on each iteration, "remove" pointer is increased by 1).
